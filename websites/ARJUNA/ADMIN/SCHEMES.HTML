Schemes

Publication organisation scheme

This scheme describes to naming and organising of the files which make up publications on the web site. In particular it specifies:

  • Publications entry points: The naming of files which form the entry points publications.
  • Organisation of multi-part publications: The structuring of files which makes up a publications.

Publications entry pointer

All publication entry points are place with in the same directory and named using the following scheme:

<publication name>.<format postfix>
or
<publication name>_<format sub-type>.<format postfix>

For example: p023.ps, p012_A4.ps, p045.html and p066_MP.html

  • publication name: The name of the publication, for example, p023 for the 23rd publication on the web site.
  • format sub-type: The name of a format sub-type, for example, USL and A4 for US Letter and European A4 postscript respectively,and MP for multi-page HTML.
  • format postfix: The postfix of the format of the publication, for example, ps for postscript, html for HTML and ps.gz for gziped postscript.

Organisation of multi-part publications

For publication which comprise of multiple files (such as html) the files, except the publications entry point, should be placed in a directory in the same directory as the publications entry point which is named using the following scheme:

<publication name>_<format postfix>
or
<publication name>_<format sub-type>_<format postfix>

For example: p045_html and p066_MP_html

publication name, format sub-type and format postfix are as defined in the previous section.

An example of the resulting structure, for a publication called p053 is given below:

p053.ps                     <entry point for postscript>
p053.ps.gz                  <entry point for gziped postscript>
p053.html                   <entry point for html>
p053_html/img1.gif
         /img2.gif
p053.html.tar               <entry point for tared html>
p053_MP.html                <entry point for multi-page html>
p053_MP_html/img1.gif
            /img2.gif
            /page1.html
            /page2.html
p053_MP.html.tar           <entry point multi-page tared html>

HTML publications maintenance

Publications which are provided as HTML should be maintained to ensure:

  • The publications does not contain "broken"
  • Browsers can display/print the publications correctly

If the publication was originally provided in HTML then the original should also be provided. The "maintained" publications should be indicated as such.
