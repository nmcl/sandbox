Thread, Thread_Type - runtime thread implementation

SYNOPSIS

       #ifndef THREAD_H_
       #   include <ClassLib/thread.h>
       #endif

       class Thread : public Resource
       {
       public:
            virtual void Suspend () = 0;
            virtual void Resume () = 0;
            virtual void Body () = 0;

            virtual long Current_Thread () const = 0;

            virtual long Identity () const;

            static Thread *Self ();

       protected:
            Thread ();
            virtual ~Thread ();

            long thread_key;
       };

       #define Thread_Type 'thread implementation'

DESCRIPTION

       This  class  defines a template for all thread implementa-
       tions used to provide an active  object  abstraction.  The
       actual  thread  implementations  form  classes  which  are
       derived from this class, and are hidden by the Thread_Type
       CPP  macro  definition,  which  is defined at system build
       time.  All classes which  require  a  threaded  capability
       should be derived from this Thread_Type "class" for porta-
       bility reasons, rather than directly from a  given  thread
       implementation.

       The  Suspend method is used to suspend the thread to which
       it is applied.

       The Resume method resumes the thread it is applied to.

       Body is the main active object code which is executed when
       the threaded object is created.

       Current_Thread  returns  the  identity  of  the  currently
       active thread, or -1 if an error occurs.

       Identity returns the identity of the threaded object it is
       object.

       thread_key is used  to  uniquely  identify  each  threaded
       object  within  the  system.  It must be set by the actual
       (derived) thread class.
